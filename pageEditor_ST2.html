Splash screen

Technical Description

Action Impact Simulator (AIS) is a web application that can be used to play the role of environmental advocate with SDG 14 on the client side and based on HTML5, CSS3, and plain JavaScript. The interface has a CSS Grid-based design, displaying the so-called action cards, which users can interact with. A card is associated with a JavaScript object with its IDs, a point value, and a descriptive metadata.

The primary section of the application is implemented with the help of a JavaScript Set, which tracks user preferences. This results in Document Object Model (DOM) being dynamically re-calculated to calculate a so-called impact score and update the background images dynamically with the help of the src and the opacity properties as a means of providing visual feedback.

Web accessibility is also a priority to the file through adding ARIA roles, screen reader live regions, and event listeners to keyboard navigation (Enter/ Space keys). This ensures that it can be used by everybody and they enjoy.

Accessibility

Action Impact Simulator (AIS) also focuses on the digital inclusion process and incorporates multiple important accessibility features to provide the users with various needs with a seamless experience.

A technical implementation of the interface is based on semantic HTML and the ARIA (Accessible Rich Internet Applications) attributes (like role=listitem, aria-live=polite) that enable screen readers to notify the user about the changes in the impact score and level in real-time but without breaking the flow.

To users without a mouse, the cards are given a tabindex of 0, and the cards are fully accessible, and custom JavaScript event handlers can be used to interact with the Enter or Space keys. In addition to that, the application offers visible and readable feedback, with dynamic background images having a descriptive alt text that makes it possible to see the fonts and labels as highly contrasted, making it visible that the mission of SDG 14 can be followed by any person and used.

Action Impact Simulator

Technical Description

The Action Impact Simulator (AIS) is a robust and client-side web application that is written in HTML5, CSS3, and vanilla JavaScript to gamify SDG 14 environmental advocacy. The interface is made up of a CSS Grid layout to display interactive action cards, which are programmatically created out of a JavaScript object of unique IDs, point values, and descriptions.

The business logic of the application has a JavaScript Set to store user selections, which cause a real-time update to the Document Object Model (DOM) to compute a cumulative “impact score” by the use of point thresholds. Moreover, the page has a dynamic background system with the help of which the src and the property of the opacity give the images in order to replace each other, namely, to the dull waves, the colorful coral reef, in order to inform the user of their progress immediately.

The file is designed to be inclusive and therefore includes web accessibility standards through the use of ARIA roles, a tabindex to navigate the site with a keyboard, and aria live region to announce changes in the score to screen readers.

Marine Biodiversity Page

Technical Description

Marine Biodiversity content page is a complex, data-driven document that was created with the emphasis on the information architecture structure and responsive web design. Technically, the page has been designed using CSS Grid with a grid-template-column: 220px 1fr grid-template-column arrangement to construct a two-column desktop layout that would smoothly convert to a one-column stack in mobile devices through media queries.

A position sticky side navigation menu will be implemented to improve the user experience by adding a consistent, clickable table of contents that allows one to jump quickly between sections focused on a specific topic, such as the Species and Threats.

Additionally the page incorporates more sophisticated visual information elements such as high-resolution extrinsic image resources and explanatory figure elements to offer a scientific background. The developer used CSS custom properties (variables) to a centralised colour and typography system, ssibility-oriented HTML5 tags aria-label and semantic article and section labels to make sure that the information can be easily perceived and navigated by every user.

Challenges and Lessons Learned

The main technical difficulty on the Marine Biodiversity and Action Impact Simulator pages was to transform complicated ecological information to an easy user interface. In the case of the biodiversity page, the challenge was to use a CSS Grid-based system that ensured there was a functional, sticky side-navigation menu and that still had to handle high-resolution scientific images, depending on the breakpoint. The hurdle in the simulator moved to the JavaScript logic; the correct handling of the events to ensure that the Set object properly followed the choices made by unique users to refresh the "Impact Score" in real-time necessitated a strict event management to avoid the calculation of a point twice.

One of the biggest development lessons was the need to have a solid accessibility instead of seeing it as a finishing touch. Incorporating ARIA roles, tabindex to make work with the keyboard, and aria-live regions to update the scores dynamically demonstrated that inclusive design helps in a greater degree to improve the UX of all users. The semantic HTML5 elements such as article and section developed the document structure to be inherently more navigable to assistive technologies, which once again supported the objective of the project to raise awareness among the largest audience possible regarding SDG 14.

The process of development also emphasized the influence of the visual feedback in influencing the environmental advocacy. Changes within the simulator, such as the background moving past basic waves to a living coral reef, as the user gains the score, showed that gamification can cause abstract objectives, such as carbon sequestration, to become more concrete. Moreover, the application of CSS custom properties also enabled the creation of a centralized system of design, and this has taught the importance of having maintainable code to work with several content-heavy pages that need a uniform visual representation.

Compliance

The pages designed in the project OceanAct- Marine biodiversity, Action Impact Simulator and Splash Screen meet the acceptance use Policy (AUP) and Security Policy in that all the content and technical implementations support the network mission which is the education and research. According to Janet policies, users are required to make sure that their hosted content is legal, not commercial (in a personal/student perspective), and does not impair the integrity of the network. I ensure that these are strictly followed on my pages with the specific SDG 14 (Life Below Water) which has evidence-based educational content and has no outlawed content like unsolicited advertising, spam, or offensive text. Citing reliable sources such as the IUCN RedList and WWF, the pages also complete the Janet requirement of the accuracy of information as well as its suitability to the needs of an academic community.

Security and technical compliance wise the pages are to be safe to others as stipulated by the Janet Security Policy. The Action Impact Simulator is written in pure, client-side vanilla JavaScript without any outside libraries or invasive scripts that may trigger the proactive vulnerability scanning of Jisc. This will avoid exposing the greater institutional network to the threat of malware or security vulnerabilities being introduced through this. Moreover, semantic HTML5 and ARIA roles are used to meet the 2026-compliant level of digital accessibility (as in WCAG 2.1 Level AA) which is slowly becoming a regulatory requirement to any publicly-served academic content on a UK academic network.

Lastly, the pages are also honest to the Janet Connection Policy, in that they are hosted within the accepted institutional structure of student coursework. It does not seek to use the infrastructure connected to Janet in private business or does not seek to circumvent the network security measures (e.g., by transmitting data unauthorised). Accountability and transparency in the pages are ensured through the addition of a Page Editor link and validation credentials to the footer, thus enabling the network administrators to establish the identity of the content creator, which is one of the elements of the "appropriate technical and operational arrangements" that organizations should be able to demonstrate to Jisc.
