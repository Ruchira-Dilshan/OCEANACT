Home page

Technical Description

The Home page uses an external CSS stylesheet to maintain a coherent design system, utilizing custom CSS variables (e.g., `--primary`, `--secondary`) for colours and typography. Layout is structured using semantic HTML tags (`<header>`, `<main>`, `<section>`, `<article>`). Distinct sections, such as "Why life below water matters," use CSS Grid (`display: grid`) mapped through a `.card-grid` to display responsive cards. A key interactive element is the `.gallery-tile` system, using CSS hover transform properties (`transform: scale()`) and linear gradients with transparent overlays to surface tile labels on hover.

Accessibility

Accessibility was actively incorporated through precise ARIA labels (`aria-label`, `aria-hidden`), especially on the logo icons and navigation lists. Non-decorative images have descriptive `alt` attributes. Standard semantic landmarks like `<nav>`, `<main>`, and `<footer>` were utilized to ensure screen reader compatibility. Contrasting colour values were evaluated to ensure text is legible against the varied gradient backgrounds of the gallery tiles.

Gallery Page

Technical Description

The Gallery page implements a dynamically generated thumbnail grid. An underlying JavaScript array of objects maps image metrics to HTML `<article class="thumb-card">` nodes using document creation methods. A significant interactive feature is the modal viewer component (`#modal-overlay`). Clicking any grid node changes the modal style attributes to `display: flex` and pushes contextual content attributes into internal target nodes. Furthermore, local `<select>` widgets enable dynamic mutation of aesthetic variables (background colours and font families) through attached event listeners rendering localised adjustments instantly upon change.

Accessibility

Navigating the complex gallery architecture maintains profound keyboard and screen reader accessibility via explicit ARIA property administration. Visually injected JS cards employ `tabindex="0"`, natively handling `keydown` EventListeners identifying "Space" and "Enter" inputs equivalently aligned to mouse click actions. Active popups declare `aria-modal="true"`, ensuring screen readers recognize context transitions and logically bind focus onto functional internal closure buttons (`#modal-close`).

Content Page

Technical Description

The "Ocean Pollution" Content Page leverages a structurally readable flow centered inside a `max-width: 900px` main wrapper. Distinct to this environment is a specialized sticky internal navigation component (`position: sticky`), enabling efficient multi-sectional anchor navigation. We also used `scroll-margin-top` to ensure section titles perfectly align below the adjacent headers dynamically. We interspersed the informative text with `.stat-card` graphical modules displaying flexible wrap conditions mapped within local grid scopes `display: grid; grid-template-columns: repeat(auto-fill, minmax(180px, 1fr))` seamlessly integrating layout and typography.

Accessibility

Clear responsive topography is managed continuously through clamped properties (`clamp()`) mapping text scales effectively based on the viewport. Navigational jump links communicate absolute context through distinct labeled parameters, preventing confusion. Furthermore, semantic ordering establishes solid logical relationships naturally parsed successfully throughout all `<h2>`, `<h3>` headings. A visually apparent "Go to top" anchor with an `aria-label` provides a simple way to traverse back through the content flow.

Challenges and Lessons Learned

A notable challenge emerged whilst engineering robust interactive DOM nodes within the Gallery page without compromising accessibility. Ensuring injected DOM fragments could handle focus correctly needed thoughtful listener attachments (`keydown` and `click`) alongside careful evaluation of ARIA structures (`role="listitem"`). Overcoming these hurdles emphasized the magnitude of separating content, style, and logic systematically. Another challenge was managing CSS variables seamlessly beside page-specific styles, which was resolved by strictly scoping local configurations to avoid overwriting the global `.css` baseline.

Compliance

Compliance across pages actively adheres to criteria necessitated by the Janet Acceptable Use Policy framework. The digital deployment comprises educational, non-commercial knowledge respecting standard intellectual property conventions. Specifically, all media utilizes natively licensed, royalty-free assets (sourced correctly through Unsplash), thereby ensuring there is no infringement of copyright or unauthorized transmission of protected material across network borders.
